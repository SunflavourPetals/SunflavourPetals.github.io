    C++ 完美转发的实现

    本文将介绍 std::forward 怎样做到“完美转发”及代码实现。
    讲完美转发前需要先了解转发引用和引用折叠。

    转发引用

    在函数模板中，为了可以根据传入参数的值类别，使形参的类型为相应的左值引用/右值引用，我们通常使用转发引用，也称为万能引用： 

    template <typename T>
void foo(T&& arg) {}

    对 C++20 新增的简写函数模板也是如此： 

    void foo(auto&& arg) {}

    下面的调用中： 

    foo(1);
int num{ 1 };
foo(num);

    对于 foo(1)，形参 arg 的类型是 int&&；对于 foo(num)，形参 arg 的类型是 int&。这都是用引用折叠来实现的。

    虽然 T&& 看上去是右值引用的形式，但是需要考虑 T 是引用类型的别名或是需要被推导的类型的情况，请看“引用折叠”。

    引用折叠

    C++ 不允许引用的引用，如 int & & r = num; int & && r = 1;……
    这些都无法通过编译。

    但是当 T 是引用类型的类型别名时，使用 T& 和 T&& 将发生引用折叠。 

    using T = int&;
using U = int&&;
int num{};

T&  r1 = num;
T&& r2 = num;
U&  r3 = num;
U&& r4 = 0;

    引用折叠的规则：&& && 折叠成右值引用，所有其他组合均折叠成左值引用。

    y运用上述规则进行推导，得出：

    标识符 类型声明 替换类型别名后 引用折叠后的类型
    r1 T& int& & int&
    r2 T&& int& && int&
    r3 U& int&& & int&
    r4 U&& int&& && int&&

    除了类型别名，使用 模板参数&& 和 auto&& 也能发生引用折叠： 

    template <typename T>
void foo(T&& arg) // T&& 将发生引用折叠
{
    std::vector<int> vec{ 1, 2, 3 };
    for (auto&& e : vec) // auto&& 同样会发生引用折叠
    {
        std::cout << e << ' ';
    }
}

    至于模板参数T和auto如何被推导为引用类型而引发引用折叠，请看下节。

    模板参数的类型推导

    还是以 foo 函数模板为例，为了使 foo 接收拥有不同值类别的参数并且使形参为相应引用，如何推导模板参数 T 的类型？ 

    template <typename T>
void foo(T&& arg) {}

void test()
{
    int num = 1;
    foo(1);
    foo(num);
}

    foo(1) 中，实参 1 是纯右值，形参 T&& arg 中，arg是引用而且只能是右值引用(因为无法将左值引用绑定到右值)，T 被推导为 int，T&& 刚好是 int&&；
    foo(num) 中， 实参 num 是左值，形参 T&& arg 中，arg是引用而且只能是左值引用(因为无法将右值引用绑定到左值)，T 被推导为 int&，T&& 则是 int& &&，引用折叠为 int&；

    此外，auto 使用和模板参数一样的推导规则。

    类型推导和引用折叠双剑合璧，就成为转发引用。

    转发引用使用引用折叠，根据实参的值类别使形参有左值引用或右值引用，那么为什么需要完美转发呢？请看下节。

    引用

    上节提到左值引用只能绑定到左值，右值引用只能绑定到右值。值类别是个需要严肃讨论的东西，但是本文主题不在于此，关于值类别，不同于传统 C 语言，C++11 给出了更加完善的定义，参考C++ Reference(中文)。

    整型、浮点型、字符、布尔字面量都是纯右值(字符串字面量是左值)。只能用右值引用去引用上述为右值的字面量： 

    int&& rref = 1;

    但是引用都是左值，包括右值引用： 

    int& lref = rref; // 例如这里将左值引用绑定到rref，绝对可以，轻易可以。

    对于函数也是如此： 

    void bar(int& lref) { std::cout << "void bar(int&)\n"; }
void bar(int&& rref) { std::cout << "void bar(int&&)\n"; }
void test()
{
    int&& rref = 1;
    int num = 1;
    int& lref = num;
    bar(num);  // num  是左值，调用 void bar(int&)
    bar(1);    // 1    是右值，调用 void bar(int&&)
    bar(lref); // lref 是左值，调用 void bar(int&)
    bar(rref); // rref 是左值，调用 void bar(int&)
    foo(rref); // 不例外，模板参数 T 被推导为 int&，引用折叠后形参类型为 int&
}

    故虽然转发引用有用，但当遇到将转发引用的参数传递给其他使用转发引用的函数模板时，无法继续保持正确的引用(左值引用或右值引用)。
    解决方案请看下节，完美转发。

    完美转发

    为了使对象成为一个右值，我们可以使用 std::move，std::move 接收一个左值或右值，并返回一个右值。

    一般 std::move 的实现： 

    // C++14
template <class T>
constexpr remove_reference_t<T>&& move(T&& arg) noexcept
{
    return static_cast<remove_reference_t<T>&&>(arg);
}

    对于 int 类型左值 num，static_cast<int&&>(num) 同样能将左值转换为右值，但是需要我们显式填入类型，std::move(num) 则不需要，更加简短方便。

    如上文所述，引用都是左值。虽然引用都是左值，但是若函数返回类型是右值引用，那么这个函数调用表达式的值类别是右值；若函数返回类型是左值引用，那么这个函数调用表达式的值类别是左值。
    (若函数返回值类型不是引用，那么函数调用表达式的值类别是右值)

    由类型推导小节，以 

    template <typename T>
void foo(T&& arg) {}

void test()
{
    int num = 1;
    foo(1);
    foo(num);
}

    为例，对于 foo(1)，形参 arg 有类型 int&&，T 为 int；对于 foo(num)，形参 arg 有类型 int&，T 为 int&。

    有以上信息帮助，我们就能分辨出哪些情况应该使用 std::move 将 arg 制作成右值，但是这还不够，我们应该有一个函数模板，根据以上信息返回左值或右值，那就是 std::forward。

    我们可以实现一个 my_forward，让它具有 std::forward 的功能。

    对于 my_forward，foo 的模板参数 T 和形参类型 decltype(arg) 是有用的信息；
    我们选用 T 作为有效信息就已足够，选用它的原因是使用它足够方便；
    my_forward 应该是一个模板，令模板参数列表为 <typename T>；
    使用引用折叠来处理返回类型；
    由于转发所接收的参数通常是引用，对形参，应为左值引用；
    返回语句同样需要引用折叠配合 static_cast 来处理成正确的类型。

    my_forward 的实现如下： 

    template <typename T>
constexpr T&& my_forward(std::remove_reference_t<T>& arg) // arg 是左值引用
{
    return static_cast<T&&>(arg);
}

    由于 T 是我们要用的信息，所以应该显式填入模板的参数列表，而不是令其推导。 

    template <typename T>
void forward_test(T&& arg)
{
    foo(my_forward<T>(arg));
}

    成功完美转发！

    根据引用折叠的知识，我们来推演一下 forward_test 的 arg 分别有类型 int& 和 int&& 时，my_forward 的工作状况：

    arg 为左值引用时

    已知当 arg 为 int& 时，T 也为 int&，
    将 T 填入后 my_forward<T>(arg) 即是 my_forward<int&>(arg)。
    再来看 my_forward 的处理：
    返回值类型 T&& 即是 int& &&，引用折叠后为 int&，
    参数是 std::remove_reference_t<T>& arg，经处理后是 int & arg，
    返回语句 return static_cast<T&&>(arg); 即是 return static_cast<int& &&>(arg);，引用折叠后为 return static_cast<int&>(arg);，
    成功转发为左值。

    arg 为右值引用时

    已知当 arg 为 int&& 时，T 为 int，
    将 T 填入后 my_forward<T>(arg) 即是 my_forward<int>(arg)。
    再来看 my_forward 的处理：
    返回值类型 T&& 即是 int&&，
    参数是 std::remove_reference_t<T>& arg，经处理后是 int & arg，
    返回语句 return static_cast<T&&>(arg); 即是 return static_cast<int&&>(arg);，
    成功转发为右值。

    完美转发·续

    标准库还提供了为右值准备的 std::forward，以 msvc-stl为例： 

    _EXPORT_STD template <class _Ty>
_NODISCARD _MSVC_INTRINSIC constexpr _Ty&& forward(remove_reference_t<_Ty>&& _Arg) noexcept {
    static_assert(!is_lvalue_reference_v<_Ty>, "bad forward call");
    return static_cast<_Ty&&>(_Arg);
}

    可以看到为右值提供的 std::forward 不允许将右值转发为左值！
